# PRD â€” Product Requirement Document (WHAT & WHY)

**Audience:** Business stakeholder, product, designer, engineer (non-technical friendly)

Tujuan: menjelaskan *masalah, value, user, dan scope produk*.

---

## 1. Problem & Value

* Background masalah
* Problem statement
* Value proposition
* Success metrics (OKR/KPI)

Contoh metric:

* Daily active users (DAU) / MAU
* Conversion rate
* Revenue impact
* Time saved

---

## 2. User

* Target user
* Pain points user

---

## 3. Features & Scope

### Core features

* Feature list
* User flow (UI journey)

### Scope definition

* MVP scope
* Out of scope

---

## 4. Product Risks

Untuk awareness stakeholder:

* Requirement belum stabil
* Integrasi belum pasti

Gunakan format:

```
Risk:
Impact:
Potential:
Mitigation:
```

---

## 5. Constraints

Untuk awaraness engineer:

* Budget terbatas
* Target release
